Task Control Polling vs. Interrupts

How does the MCU know when an input happens?

1. Software Polling

  • How it works: A continuous loop while(1) regularly checks the input.
  • Pros: Simple to write.
  • Cons: Wastes power. Inefficient for rare events (e.g., a lift button).

2. Hardware Interrupts

  • How it works: Hardware triggers the MCU only when an event occurs. The MCU can sleep otherwise.
  • ISR (Interrupt Service Routine): The code that runs when the interrupt fires. Must be fast!
  • Context Saving: The MCU saves its state to the stack, runs the ISR, and loads the state back.
